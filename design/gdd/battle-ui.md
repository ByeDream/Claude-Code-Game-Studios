# Battle UI (战斗UI)

> **Status**: Designed
> **Author**: User + Claude
> **Last Updated**: 2026-03-29
> **Implements Pillar**: 养成爆发, 构筑智慧

## Overview

Battle UI 负责战斗场景的所有视觉信息展示，覆盖战前编排、战斗进行中和战后结算三个阶段。战前阶段展示敌方阵容预览和我方站位编排界面；战斗中实时渲染武将动作、伤害数字、技能特效、状态图标和血条；战后展示战斗回顾和宝箱奖励。UI 设计原则是"信息清晰、操作极简"——自走棋战斗中玩家几乎不操作（仅军师技），所以 UI 的核心任务是让玩家**看懂发生了什么**而非**做什么操作**。

## Player Fantasy

**「端坐帅帐，纵观全局。」**

战场一览无余——你的武将和敌方武将面对面站成两排，血条、状态、技能冷却一目了然。当你的关羽挥出青龙偃月刀，伤害数字从敌人头顶弹出，你能清楚地看到装备加成和羁绊效果在发挥作用。行动顺序条告诉你下一个动的是谁，让你在关键时刻精准释放军师技。战斗结束后的 MVP 统计和宝箱奖励是对你构筑智慧的最终评价。

**Pillar 对齐**:
- **养成爆发**: 伤害数字、属性加成、技能效果让每一次强化都肉眼可见
- **构筑智慧**: 战前编排界面和战斗中的信息展示帮助玩家验证构筑决策

## Detailed Design

### Core Rules

#### 1. Three Phases (三阶段)

| 阶段 | 玩家操作 | UI 核心任务 |
|------|---------|------------|
| **战前编排** | 拖拽排列站位、查看敌方、选军师技 | 提供信息 + 操作界面 |
| **战斗进行** | 观战（+ 军师技 VS+） | 实时信息展示 |
| **战后结算** | 查看结果、宝箱三选一 | 结果展示 + 奖励交互 |

#### 2. Pre-Battle Phase (战前编排)

**屏幕布局**：

```
┌──────────────────────────────────────┐
│           敌方阵容预览                │
│  [敌1] [敌2] [敌3] [敌4] [敌5]      │
│  (名称、Tier、HP条、技能图标)         │
├──────────────────────────────────────┤
│           战场区域（空）              │
├──────────────────────────────────────┤
│           我方编排区                  │
│  [位0] [位1] [位2] [位3] [位4]      │
│  (拖拽武将到位置)                     │
├──────────────────────────────────────┤
│  [替补席武将列表]    [军师技选择]      │
│                      [开战按钮]       │
└──────────────────────────────────────┘
```

**信息展示**：

| 信息 | 展示方式 | 交互 |
|------|---------|------|
| 敌方武将 | 立绘缩略图 + 名称 + Tier 框 + HP 条 | 点击查看详情（属性/技能） |
| 敌方Boss 特殊信息 | Boss 名称 + 状态抗性预览 | 点击查看 |
| 我方站位 | 5 个槽位，可拖拽武将进出 | 拖拽排列 |
| 武将详情 | 点击武将卡 → 弹出属性/技能/装备面板 | 点击 |
| 替补席 | 横向列表，未上场武将 | 拖拽到站位 |
| 军师技 | 可用军师技图标（VS+ 阶段） | 点击选择 |
| 羁绊预览 | 当前阵容激活的羁绊列表 | 自动更新 |
| 开战按钮 | 底部中央 | 点击开战 |

#### 3. Combat Phase (战斗进行中)

**屏幕布局**：

```
┌──────────────────────────────────────┐
│ [行动顺序条]                          │
│ ○○○●○○○○○○  (当前行动者高亮)         │
├──────────────────────────────────────┤
│                                      │
│  敌方:  [敌1] [敌2] [敌3] [敌4] [敌5]│
│         HP条 + 状态图标               │
│                                      │
│                                      │
│  我方:  [位0] [位1] [位2] [位3] [位4]│
│         HP条 + 状态图标               │
│                                      │
├──────────────────────────────────────┤
│ [回合数]              [军师技按钮]     │
│ [倍速按钮: 1x/2x/3x] [跳过按钮]     │
└──────────────────────────────────────┘
```

**实时信息**：

| 信息 | 位置 | 更新频率 |
|------|------|---------|
| 武将 HP 条 | 武将头顶 | 每次受伤/治疗 |
| HP 数值 | HP 条内或旁边 | 每次受伤/治疗 |
| 状态图标 | HP 条下方，小图标排列 | 状态施加/移除时 |
| 状态剩余回合 | 状态图标上的小数字 | 每回合 |
| 伤害/治疗数字 | 目标头顶弹出 | 每次伤害/治疗 |
| 暴击标识 | 伤害数字放大 + 特殊颜色 | 暴击时 |
| 行动顺序条 | 屏幕顶部 | 每次行动后 |
| 当前行动者 | 行动顺序条高亮 + 武将边框发光 | 实时 |
| 技能释放名称 | 屏幕中央短暂弹出 | 技能触发时 |
| 回合数 | 左下角 | 每回合 |
| 军师技按钮 | 右下角（VS+ 阶段） | 可用时亮起 |

**动画层级**（优先级从高到低）：

| 优先级 | 动画 | 说明 |
|--------|------|------|
| 1 | 军师技全屏演出 | 玩家手动触发，最高优先 |
| 2 | 武技触发 | 屏幕震动 + 武将特写 |
| 3 | 主动技能 | 技能名称弹出 + 特效 |
| 4 | 武将阵亡 | 倒下动画 + 灰度化 |
| 5 | 普通攻击 | 攻击动作 + 伤害数字 |
| 6 | 状态施加/移除 | 图标出现/消失 |
| 7 | DoT/HoT tick | 小数字弹出 |

#### 5. Action Order Bar (行动顺序条)

显示当前回合所有存活武将的行动顺序（按 SPD 排序）：

```
[诸葛亮] [赵云] [敌3] [关羽] [敌1] [张飞] [敌2] [敌4] [敌5]
         ↑ 当前行动者高亮
```

- 我方武将用蓝色边框，敌方用红色
- 当前行动者放大高亮
- 已行动的向左移出，未行动的向右排列
- 阵亡武将从顺序条中移除

#### 6. Damage Numbers (伤害数字)

| 类型 | 颜色 | 大小 | 动画 |
|------|------|------|------|
| 物理伤害 | 白色 | 标准 | 向上弹出并淡出 |
| 技能伤害 | 蓝色 | 标准 | 向上弹出并淡出 |
| 暴击伤害 | 橙色 | 放大 1.5x | 向上弹出 + 震动 |
| 治疗 | 绿色 | 标准 | 向上弹出 |
| 中毒/燃烧 tick | 紫色/橙色 | 缩小 0.8x | 缓慢上浮 |
| 护盾吸收 | 灰色 | 标准 | 向上弹出 |

#### 7. Speed Controls (播放控制)

| 控制 | 功能 | 快捷键 |
|------|------|--------|
| 1x 速度 | 默认播放速度 | 1 |
| 2x 速度 | 双倍速 | 2 |
| 3x 速度 | 三倍速 | 3 |
| 跳过 | 直接跳到战斗结果（无动画） | Space |

- 速度设置跨战斗保持（记住上次选择）
- 跳过后直接进入结算阶段

#### 8. Responsive Design

| 屏幕宽度 | 布局调整 |
|---------|---------|
| ≥1280px | 标准布局，所有信息完整展示 |
| 1024-1279px | 替补席折叠，点击展开 |
| <1024px | 不支持（PC 优先，最低 1024px） |

### States and Transitions

| State | Entry | Exit | UI Mode |
|-------|-------|------|---------|
| **Pre-Battle** | 进入战斗节点 | 点击「开战」→ Combat | 编排模式：可拖拽、可查看 |
| **Combat** | 开战 | 一方全灭/超时 → Post-Battle | 观战模式：仅军师技可操作 |
| **Post-Battle** | 战斗结束 | 奖励完成 → 返回地图 | 结算模式：宝箱选择 + Boss 额外掉落 |

### Interactions with Other Systems

| System | Direction | Interface |
|--------|-----------|-----------|
| **Battle Engine** | Engine → UI | 推送 `BattleEvent` 流（攻击、技能、伤害、死亡、状态等） |
| **Hero System** | UI 读取 Hero | 武将立绘、属性、技能信息展示 |
| **Equipment System** | UI 读取 Equipment | 装备信息（战前查看） |
| **Bond System** | UI 读取 Bond | 战前编排时显示已激活羁绊 |
| **Status System** | UI 读取 Status | 战斗中状态图标和剩余回合 |
| **Loot/Rewards** | UI 读取 Loot | 战后宝箱展示和三选一交互 |
| **Enemy System** | UI 读取 Enemy | 敌方阵容信息、Boss 抗性预览 |
| **Event System** | Event → UI | 战斗节点进入/退出 |

**接收的事件流**（来自 Battle Engine）：
```typescript
// UI 监听 BattleEvent 流并逐个渲染
interface BattleEvent {
  type: 'attack' | 'skill' | 'damage' | 'heal' | 'death' | 'status_apply' | 'status_expire' | 'status_tick' | 'martial_art' | 'advisor_skill' | 'boss_phase' | 'round_start' | 'round_end';
  source: string;
  target: string[];
  value?: number;
  isCrit?: boolean;
  statusId?: string;
  skillName?: string;
}
```

## Formulas

Battle UI 是纯展示层，不包含游戏逻辑公式。以下是渲染相关计算：

### Damage Number Position

```
numberX = target.positionX + randomOffset(-10, 10)
numberY = target.positionY - HP_BAR_OFFSET
```

### Animation Timing

```
attackAnimDuration = BASE_ATTACK_ANIM / speedMultiplier
skillAnimDuration = BASE_SKILL_ANIM / speedMultiplier
```

| Variable | Value | Description |
|----------|-------|-------------|
| BASE_ATTACK_ANIM | 400ms | 普通攻击动画时长 |
| BASE_SKILL_ANIM | 600ms | 技能动画时长 |
| BASE_MARTIAL_ANIM | 800ms | 武技动画时长 |
| BASE_ADVISOR_ANIM | 1200ms | 军师技全屏演出时长 |
| speedMultiplier | 1/2/3 | 当前播放速度 |

### HP Bar Width

```
hpBarFillRatio = currentHP / maxHP
hpBarWidth = HP_BAR_MAX_WIDTH * hpBarFillRatio
```

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 同一帧多个伤害数字重叠 | 错位展示（向上依次偏移） | 避免数字重叠不可读 |
| 极高伤害数字（4位数以上） | 数字自动缩小字号 | 不超出显示区域 |
| 10 个状态图标同时存在 | 最多显示 6 个图标，超出部分显示"+N"，hover 查看全部 | 防止视觉混乱 |
| 3x 速度下动画时长 < 16ms | clamp 到最小 1 帧（16ms） | 防止动画消失 |
| 战斗中武将全部阵亡（瞬间） | 按顺序播放阵亡动画，不跳过 | 保证玩家看到结果 |
| 点击「跳过」后战斗结束 | 跳过战斗动画，直接进入结算界面；Boss 额外掉落仍正常展示 | 跳过的是动画不是结果，掉落奖励不可跳过 |
| 战前编排：拖拽武将到已有武将的位置 | 两个武将交换位置 | 直观操作 |
| 战前编排：从站位拖回替补席 | 该站位变空 | 允许少于 5 人出战 |
| 行动顺序条中 SPD 完全相同的武将 | 随机排列，但本回合内顺序固定 | 与 Battle Engine 一致 |
| Boss 多阶段转换 | 播放阶段转换动画，暂停行动顺序条 | 视觉上明确阶段分界 |
| 网页失焦（切到后台） | 战斗暂停，回来后继续 | 防止错过关键时刻 |

## Tuning Knobs

| Parameter | Initial | Safe Range | Effect of ↑ | Effect of ↓ |
|-----------|---------|------------|------------|------------|
| `BASE_ATTACK_ANIM` | 400ms | 200-600ms | 攻击动画更慢、更有仪式感 | 更快节奏 |
| `BASE_SKILL_ANIM` | 600ms | 300-1000ms | 技能动画更华丽 | 更紧凑 |
| `BASE_ADVISOR_ANIM` | 1200ms | 800-2000ms | 军师技演出更长 | 更短促 |
| `HP_BAR_MAX_WIDTH` | 80px | 60-120px | 血条更宽、更醒目 | 更紧凑 |
| `DAMAGE_NUMBER_DURATION` | 800ms | 500-1200ms | 伤害数字停留更久 | 消失更快 |
| `MAX_VISIBLE_STATUS_ICONS` | 6 | 4-8 | 更多图标可见 | 更简洁 |
| `DEFAULT_SPEED` | 1 | 1-2 | 默认更快 | 默认标准速 |
