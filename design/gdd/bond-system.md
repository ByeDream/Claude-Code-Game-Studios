# Bond System (羁绊系统)

> **Status**: Designed
> **Author**: User + Claude
> **Last Updated**: 2026-03-29
> **Implements Pillar**: 构筑智慧, 收集之乐, 历史沉浸

## Overview

羁绊系统为阵容中符合特定条件的武将组合提供额外属性加成。分为阵营羁绊（同阵营人数越多加成越高）和历史羁绊（特定武将组合触发独特效果）。本系统是锦上添花而非核心强度来源——前期帮助玩家理解阵容搭配逻辑，后期玩家会为了更强的技能配合而主动打破羁绊组合。

## Player Fantasy

**「桃园三结义，兄弟齐上阵！」**

当你凑齐刘关张看到"桃园结义"羁绊亮起的那一刻，历史感扑面而来。当你发现"虎豹骑"、"江东双壁"等新羁绊时，会忍不住去查看还有哪些组合等待发现。但羁绊是起点而非终点——随着你理解技能配合的深度，你会学会何时遵循历史、何时打破历史。

**Pillar 对齐**:
- **历史沉浸**: 羁绊直接来源于三国历史典故，大量羁绊构成一张历史人物关系网
- **收集之乐**: 发现和触发新羁绊是持续的惊喜时刻
- **构筑智慧**: 羁绊是阵容搭配的参考但不是枷锁，玩家自由决定是否遵循

## Detailed Design

### Core Rules

#### 1. Bond Types

| 类型 | 触发条件 | 加成强度 | 数量 |
|------|---------|---------|------|
| **阵营羁绊** | 阵容中同阵营武将达到阈值 | 低（锦上添花） | 4 个（每阵营一个） |
| **历史羁绊** | 阵容中包含特定武将组合 | 中（前期有用，后期占比递减） | 20-30 个 |

#### 2. Faction Bonds (阵营羁绊)

| 阵营 | 2人 | 3人 | 4人 | 5人(满编) |
|------|-----|-----|-----|----------|
| 蜀 Shu | 全体 STR+3% | STR+5%, HP+3% | STR+8%, HP+5% | STR+10%, HP+8% |
| 魏 Wei | 全体 INT+3% | INT+5%, DEF+3% | INT+8%, DEF+5% | INT+10%, DEF+8% |
| 吴 Wu | 全体 SPD+3% | SPD+5%, DEF+3% | SPD+8%, DEF+5% | SPD+10%, DEF+8% |
| 群 Qun | 随机属性+3% | 随机属性+5% | 随机属性+8% | 随机属性+12% |

- 加成为百分比，写入 `bondModifier`
- 阵营偏向与设定一致（蜀=攻击/血量、魏=智力/防御、吴=速度/防御、群=混沌随机）
- 可同时激活多个阵营加成（3蜀+2吴 = 蜀3人加成 + 吴2人加成）
- **数值刻意偏低**：满编 5 人同阵营的总加成约 10-18%，不构成碾压优势

#### 3. Historical Bonds (历史羁绊)

历史羁绊数量较多（20-30 个），覆盖三国主要人物关系网，是历史沉浸感的重要载体。每个羁绊的效果偏低，属于"发现惊喜"而非"构筑必须"。

- 效果为**固定值或低百分比**，前期感知明显，后期因属性基数增长占比自然递减
- 部分羁绊只需组合中的部分成员（如五虎上将任意 3 人）
- 特殊羁绊可联动装备系统（如吕布+赤兔马）
- 部分羁绊包含条件性效果（如白帝托孤：刘备阵亡时触发）

#### 4. Bond Modifier Calculation

```
bondModifier[stat] = factionBonus[stat] + sum(activeHistoricalBonds.bonuses[stat])
```

所有羁绊加成叠加计算，最终写入 Hero System 的 `bondModifier`。

#### 5. Design Philosophy: 锦上添花原则

- **羁绊不应成为构筑的决定性因素**。一个没有任何羁绊但技能配合精妙的阵容，应该强于一个满羁绊但技能不搭的阵容
- **前期引导价值**：新手通过羁绊自然学会"同阵营武将放一起有好处"，降低理解成本
- **后期自由构筑**：当玩家理解了技能配合的深度后，会主动为了更强的技能组合放弃羁绊加成——这是设计预期内的玩家成长路径
- **数值天花板**：即使满阵营+多个历史羁绊全部激活，总 bondModifier 硬上限 25%
- **历史沉浸载体**：大量羁绊构成三国人物关系网，即使不追求强度，发现新羁绊本身就是乐趣

### States and Transitions

无独立状态机。羁绊在阵容变动时实时计算：

| 事件 | 触发重算 |
|------|---------|
| 武将加入/离开阵容 | 是 |
| 装备变更 | 是（装备联动羁绊） |
| 战斗开始 | 计算一次，战斗中不重算 |
| 武将 KO (战斗中) | 不重算（保持开战时的状态） |

### Interactions with Other Systems

| System | Direction | Interface |
|--------|-----------|-----------|
| Hero System | Bond 读取 Hero | 读取阵容中所有武将的 `faction`, `bondKeys` |
| Hero System | Bond 写入 Hero | 计算结果写入每个武将的 `bondModifier` |
| Equipment System | Bond 读取 Equip | 部分羁绊检查装备（如吕布+赤兔马） |
| Battle Engine | Battle 读取 Bond | 战斗开始时读取已激活的羁绊列表和效果 |
| Hero Detail UI | UI 读取 Bond | 显示当前激活/未激活的羁绊列表和进度 |

**公共接口**:
```typescript
interface BondAPI {
  recalculate(playerRoster: Hero[], enemyRoster?: Hero[]): BondResult;
  getActiveBonds(): ActivatedBond[];
  getAllBonds(): BondDefinition[];         // 全部羁绊定义（含未激活，用于图鉴/UI）
  getBondModifier(heroId: string): StatModifier;
}

interface BondResult {
  activatedBonds: ActivatedBond[];        // 当前激活的羁绊列表
  perHeroModifiers: Map<string, StatModifier>;  // 每个武将的加成
}
```

> **注意**：敌方阵容也享受羁绊加成。传入 enemyRoster 时，为敌方独立计算羁绊效果。

## Formulas

### Bond Modifier Total

```
bondModifier[stat] = min(factionBonus[stat] + historicalBonus[stat], BOND_MODIFIER_CAP)
```

| Variable | Type | Range | Description |
|----------|------|-------|-------------|
| factionBonus | float | 0.0-0.10 per stat | 阵营加成（最高满编单属性 10%） |
| historicalBonus | float | 0.0-0.15 per stat | 历史羁绊加成总和 |
| BOND_MODIFIER_CAP | float | 0.25 | bondModifier 硬上限 |

### Effective Strength Contribution

在一个典型的中后期武将（finalStat ~200）身上：
- 满阵营加成 (10%): +20 属性点 — 相当于一件 Lv.3 通用装备
- 单个历史羁绊 (5-10%): +10-20 属性点
- 满羁绊上限 (25%): +50 属性点 — 显著但不碾压

对比参考：一件名器可提供 +20 属性 + 质变级特殊效果，远超羁绊加成。
这确保了**装备和技能配合 > 羁绊加成**的优先级。

## Edge Cases

| Scenario | Behavior | Rationale |
|----------|----------|-----------|
| 同时满足多个历史羁绊 | 全部激活，效果叠加（受 25% 上限约束） | 奖励收集深度 |
| 传说变体是否触发羁绊 | 是——武圣·关羽仍然触发"桃园结义" | 传说变体是同一人物 |
| 基础版和传说变体不能同时上场 | 不影响——同名只能上一个，用哪个版本都触发 | 与 Hero System 一致 |
| bondModifier 叠加超过 25% | 裁剪到 25%（per stat） | 防止数值膨胀 |
| 阵容中 0 个同阵营武将 | 无阵营加成，不报错 | 正常情况 |
| 羁绊需要的武将在战斗中被 KO | 羁绊在战斗开始时计算一次，战中不重算 | 避免战中频繁变动影响节奏 |
| 装备联动羁绊中装备被转移 | 编队阶段实时重算，羁绊可能激活或失效 | 实时响应编队变化 |
| "官渡之战"需要曹操+袁绍同场 | 袁绍作为敌方Boss出现时也触发（特殊处理） | 增加历史感 |
| 群 Qun 阵营的"随机属性" | Run 开始时随机确定本次 Run 的加成属性，Run 内不变 | 避免每场战斗随机带来的不确定性 |

## Tuning Knobs

| Parameter | Current | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|---------|------------|-------------------|-------------------|
| `FACTION_2_BONUS` | 3% | 1-5% | 阵营搭配更有价值 | 阵营影响更小 |
| `FACTION_3_BONUS` | 5%+3% | 3-8% | 三人同阵营更强 | 差异缩小 |
| `FACTION_4_BONUS` | 8%+5% | 5-12% | 四人同阵营更强 | 差异缩小 |
| `FACTION_5_BONUS` | 10%+8% | 8-15% | 满编阵营更有吸引力 | 混编更自由 |
| `BOND_MODIFIER_CAP` | 0.25 | 0.15-0.35 | 羁绊影响力增大 | 羁绊更不重要 |
| `HISTORICAL_BOND_COUNT` | 8-10 (MVP) / 20-30 (Full) | — | 更多历史感和发现时刻 | 更快完成设计 |

## Acceptance Criteria

- [ ] 阵营羁绊在阵容中同阵营人数达标时正确激活
- [ ] 历史羁绊在所需武将全部在阵容中时正确激活
- [ ] bondModifier 正确叠加并受 25% 上限约束
- [ ] 传说变体正确触发对应羁绊（与基础版共享 bondKeys）
- [ ] 装备联动羁绊（吕布+赤兔马）正确检测
- [ ] 阵容变动时羁绊实时重算
- [ ] 战斗中不重算（KO 不影响已激活羁绊）
- [ ] 群阵营随机属性在 Run 开始时确定，Run 内一致
- [ ] 所有羁绊定义可通过配置文件加载，无硬编码
- [ ] UI 可查看全部羁绊列表（含未激活，显示缺少的武将）
- [ ] Performance: 羁绊计算在 <2ms 内完成（全阵容）
